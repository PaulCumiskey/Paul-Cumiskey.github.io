Phishing Analysis

Phishing Investigation: "Urgent Invoice" Payload Triage

Scenario: A user reported a suspicious email titled "Overdue Invoice #99283" sent to the finance department.

Technical Breakdown

  • SMTP Header Analysis: The X-PHP-Originating-Script header revealed the mail originated from a compromised web server (193.161.193.x) rather than the spoofed globalfinance.com domain.
  • Artifact Analysis: The attachment Invoice_992.zip contained invoice.js. Static analysis of the JS revealed heavy obfuscation using String.fromCharCode to hide the download URL.
  • Dynamic Sandbox Findings: Upon execution, the script spawned wscript.exe, which reached out to a known malicious C2 endpoint (hxxp://wp-content-plugin[.]ru/api/get_payload) to fetch a stage-2 executable.

Incident Response Actions

Extracted the file hash (SHA-256) and performed a global search across the EDR. Identifed two additional hosts that downloaded the zip. Performed isolation of affected hosts and implemented a perimeter block on the identified IP range.

Outcome: Threat neutralized; Zero lateral movement detected.

SIEM / Detection Engineering

Splunk: High-Fidelity RDP Brute Force Detection

Objective: Develop an alert that identifies password spraying or brute force attempts against the Domain Controller while minimizing false positives from automated service accounts.

The Refined Logic

Standard EventCode 4625 monitoring often leads to alert fatigue. I implemented a threshold-based query with time-charting to visualize the "spike" in failures.

index=windows sourcetype="XmlWinEventLog:Security" EventCode=4625 
| stats count, values(TargetUserName) as targeted_users, values(IpAddress) as src_ip by _time 
| bin _time span=5m 
| stats sum(count) as total_failures by _time, src_ip 
| where total_failures > 20
| rename total_failures as "Failed Logins (5m Window)"

Tuning & Optimization

False Positive Reduction: Identified frequent failures from SVC_BACKUP. Added a lookup table to whitelist known service accounts that trigger failures due to scheduled password rotations, reducing noise by 45%.

Traffic Analysis

Network Forensics: Emotet C2 Beaconing Pattern Analysis

Scenario: During a routine review of Zeek logs, an internal host showed unusual outbound traffic patterns on port 8080.

Forensic Investigation

  • Beaconing Identification: Used Wireshark's I/O Graph to confirm a persistent heartbeat. The host transmitted 450 bytes every 60 seconds with a +/- 2 second jitter, a classic indicator of automated C2 (Command & Control) beaconing.
  • Payload Extraction: Extracted the HTTP GET request from the PCAP. The URI contained a unique Base64 encoded string containing the host's metadata (Machine Name, OS Version) being sent to the attacker.
  • Zeek/Bro Logs: Correlated the traffic with Zeek's conn.log, which showed the connection duration was consistent with long-lived C2 sessions rather than standard web browsing.

Conclusion

The host was confirmed to be infected with an Emotet-variant downloader. I recommended a full re-image of the host and a review of the group policy settings regarding .exe and .dll execution from %AppData% folders.
